# HANDOFF REPORT TEMPLATE
**Last Updated:** 2026-04-27

---

## HOW TO USE THIS TEMPLATE

1. Specialist completes their task
2. Specialist fills out this report
3. CEO receives the report
4. CEO loads it back into the CTO session
5. CTO updates institutional memory and determines next steps

---

## TEMPLATE

```
=== SPECIALIST HANDOFF REPORT ===
Date: [YYYY-MM-DD]
Project: [Project Name]
Specialist: [Which specialist completed this]
Status: [COMPLETE / PARTIAL / BLOCKED]

TASK SUMMARY:
[One paragraph summary of what was accomplished]

DELIVERABLES:
- [List every output: files created, commits made, decisions made]

RESEARCH FINDINGS:
[Any new information discovered about platforms, tools, or context that the CTO should know]

DECISIONS MADE:
[Any judgment calls made during the task — so the CTO understands why things are the way they are]

ISSUES ENCOUNTERED:
[Problems hit, workarounds used, things that slowed progress]

INCOMPLETE ITEMS:
[Anything not finished and why]

RECOMMENDATIONS FOR CTO:
[What should happen next? Any follow-up tasks? Any system improvements?]

LESSONS FOR INSTITUTIONAL MEMORY:
[What should be permanently remembered? Routing preferences, tool behaviors, CEO preferences discovered?]
=== END HANDOFF REPORT ===
```
